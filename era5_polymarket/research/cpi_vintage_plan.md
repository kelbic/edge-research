## 0. ⚡ Блокер снят: винтажи лежат в официальных JSON-фидах графика

Найдено веб-агентом, независимо верифицировано (CONFIRMED: значения переизвлечены из зеркал,
структура и конкретные числа совпали):

- **Источник [P]**: статические FusionCharts-фиды страницы nowcasting —
  `https://www.clevelandfed.org/-/media/files/webcharts/inflationnowcasting/nowcast_month.json`
  (CPI MoM), `nowcast_year.json` (CPI YoY), `nowcast_quarter.json`. Один файл = ~157 фреймов,
  по фрейму на целевой месяц с 2013-07 по текущий; в каждом — **дневная ось винтажей**
  (значение серии в день D = nowcast as-of D) + серия «Actual …». Т.е. полный vintage-архив —
  одним запросом. Прошлая попытка (403 на WebFetch страницы) упиралась в **UA-gate WAF**:
  браузерный User-Agent → 200, скриптовый → 403. Auth не нужен.
- **Стабильность винтажей [P]**: ретроактивно не пересматриваются — значение as-of 2022-12-14
  совпало до 14 знаков между contemporaneous-скрейпом янв-2023 и копией фида от 2025-10-29.
- **Покрытие**: обновления по рабочим дням (~19–35 винтажей/целевой месяц); **последний
  непустой винтаж датирован T-3…T-5 до релиза BLS** и действует (carry-forward) до релиза.
  Все ~15 релизов окна бэктеста (2025-03…2026-06; Oct-2025 CPI не публиковался — shutdown,
  Sep-2025 вышел 10-24) покрыты.
- **Примеры (переизвлечены верификатором)**: target 2025-08: as-of 09-08 = 2.841% YoY
  (Actual 2.916 ≈ BLS 2.9%, релиз 09-11); target 2025-09: as-of 10-21 = 2.988% (Actual 3.013);
  MoM target 2024-06: финальный винтаж +0.068% при Actual −0.056%.
- **Egress-оговорка**: из текущей research-сессии clevelandfed.org (и web.archive.org)
  блокируются прокси — **фид качается с обычной машины**:
  `curl -A "Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/126.0 Safari/537.36" -o nowcast_year.json "https://www.clevelandfed.org/-/media/files/webcharts/inflationnowcasting/nowcast_year.json"`.
- **Зеркала с данными в руках (fallback/bootstrap)**: `nthoang84/cleveland-fed-nowcasts`
  (полная копия nowcast_year.json 2013-07…2025-10 + извлечённый CSV дневных YoY-винтажей);
  `mnem0nic7/kalshi_bot` (фикстура nowcast_month.json as-of 2026-07-06 + готовый парсер
  `nowcast_source.py`); `subsetsio/connectors` (ежедневное перекачивание всех трёх фидов).
  Зеркалами можно прогнать 2025-часть бэктеста немедленно; 2026-хвост — после скачивания
  живого фида.

Парсер-каветы (из верификации, зафиксировать в коде): (а) метки дат оси — только MM/DD,
год выводить из target-месяца (январские хвосты декабрьских фреймов переваливают год);
(б) метка даты «Actual» — следующий слот оси, НЕ дата релиза BLS (не использовать как
release-date); (в) «as-of T-1» = carry-forward последнего винтажа (сам фид точки T-1 обычно
не содержит) — это допущение фиксируем явно.

## 1. Дизайн бэктеста: минимальный патч `scripts/backtest_cpi.py`

Единственная точка замены — функция `nowcast(cpi, kind_time, ym)` (строки ~70–83), выход
`(mu, sigma)` дальше идёт в `bucket_prob` без изменений. Discovery/parse/entry_price
(VWAP реальных трейдов в [endDate−3d, endDate−1d], YES-нормировка)/PnL/отчёт — не трогаем.

1. `load_cleveland_vintage(path)` → `{target_ym: [(as_of_date, yoy_nowcast), …]}` из
   nowcast_year.json (+ отдельно MoM из nowcast_month.json для вторичного среза).
   Снапшот фида коммитим в `data/` с датой скачивания (реплеябельность).
2. Проброс as-of: `nowcast(..., asof)` где **asof = endDate − 3d** (нижняя граница VWAP-окна) —
   сигнал = последний винтаж с датой ≤ asof ⇒ сигнал строго предшествует всем ценам входа,
   lookahead исключён по построению (консервативнее carry-forward до T-1).
3. `sigma` = историческая RMSE финального пре-релизного винтажа vs actual, посчитанная
   expanding-window **только на релизах строго ДО целевого** (no-lookahead; ожидание ~0.1pp
   MoM-эквивалент), floor 0.0005.
4. **Primary срез = YoY-only** (pre-registered рационал из лога 2026-07-06: perfect-foresight
   YoY +15.6%/+8.2%/hit 80%, MoM −4.4% даже при полном предвидении — MoM исключён как
   бесперспективный при любом сигнале; репортим его вторичной строкой для полноты).
5. Подвыборки для правила стабильности (то же «time period × market sub-family»):
   год (2025 / 2026) × тип бакета (BUCKET vs GT/LT).
6. Диагностика в отчёт: сравнение трёх сигналов на одинаковой выборке — naive-seasonal
   (базлайн −), vintage-nowcast (тест), perfect-foresight (потолок +15.6%) — тест обязан лечь
   между; если vintage ≈ perfect, это красный флаг lookahead-бага, а не успех (урок kelbic:
   «невозможно-хороший результат = баг»).

## 2. Правило вердикта (без изменений, цитата pre-registration @ exploration_log.md)

«GO if mean PnL/trade ≥ +3% net of spread AND median ≥ 0 AND positive across ≥2 independent
subsets (time period × market sub-family); else NO». Плюс условие разморозки капитала из
заморозки 2026-07-06 — то же самое. Пороги в `config.yaml` (`verdict_rule`) не правятся.
Прогноз автора (фальсифицируемый, фиксируется здесь ДО прогона): результат ляжет между
naive (~0 gross) и perfect (+15.6% mean YoY); GO-порог возьмёт только если преимущество
Cleveland-модели над сезонкой (~0.1pp vs ~0.2pp MoM-вход YoY) конвертируется в устойчивый
edge на её же бакетах — вероятность оцениваю < 50%.

## 3. Ёмкость и честная экономика (уже замерено, из `data/backtest_cpi_results.csv`)

Lifetime-объём YoY-бакетов ~$0.6M/релиз (медиана); 3-дневное пре-релизное окно — доля этого;
правило «не быть >5–10% объёма окна» ⇒ **~$3–6k разворачиваемого нотионала на релиз**,
~12 релизов/год. При выполненном GO (+3–6% net) ожидание — **порядка $1–2k/год**. Это
согласуется с pre-registered E4 («$2–10k book exists but $10k doesn't move the top of book»)
и с параметрами config.yaml (capital_total_usd 10000, max_position_per_market_usd 200).
**Вывод честно: даже при GO это ретейл-опцион/побочный доход, не бизнес** — поэтому приоритет 3
в ресурсном плане и жёсткий кап инженерного времени.
