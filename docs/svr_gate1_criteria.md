# Pre-registration: Gate 1 SVR-Ethereum — realized net через fork-replay

Дата фиксации: 2026-07-06. Коммит этого файла — точка pre-registration: порог
вердикта, правило выборки и маршрутная методология зафиксированы ЗДЕСЬ, ДО
первого замера квот. Хэш — в отчёте. Контекст (установлен в Gate 0, не
переизмеряется): prepare-tooling на upper-bound марже; оговорка — gross по
ценам AaveOracle = верхняя граница, realized net ниже, сильнее всего в
стресс-хвосте, где маржа и живёт.

## Вердикт-критерий (без «вероятно» — число против порога)

- **GO (→ pilot-ready кандидат, решение пользователя)**: медианный
  realized-net-margin по выборке (подмножество (б), см. ниже) ≥ **40%**
  oracle-priced gross, И p25 выборки (б) остаётся положительным после газа.
- **NO**: медиана (б) < 40%, ИЛИ p25 (б) ≤ 0, ИЛИ realized net на стресс-хвосте
  < издержек участия ($500/мес консервативный порог из Gate 0).
- Порог 40% — из блока чувствительности Gate 0 (знак консервативного EV
  переворачивается ниже ~40% реализации oracle-priced маржи на p25-входе).
  Другого порога не вводим.
- «На грани» (медиана в пределах ±5 п.п. от 40%): вердикт «пограничное,
  расширить выборку» — НЕ смягчённая формулировка, а обязательство добрать n.
- realized-net-margin% = realized_net / gross_oracle, где
  realized_net = swap-proceeds − погашенный долг − газ(факт из receipt)
  − SVR-бид(факт из рефанда); слиппедж сидит в swap-квоте.

## Правило выборки (фиксируется до просмотра квот)

Источник — кэш Gate 0 `data/svr_liquidations.json.gz` (тот же скан; фингерпринт
выигранных аукционов был верифицирован на живом блоке до скана — Gate 0).
Пороги ниже опираются ТОЛЬКО на уже опубликованные в Gate 0 распределения
(медиана gross $569, p75 $5.3k, mean $18.7k):

- **Стресс-хвост, не типовые аукционы**: событие входит в выборку, если
  oracle-priced gross ≥ **$20k** (заведомо > p90 опубликованного распределения).
  Типовые/медианные события НЕ мерятся и НЕ усредняются с хвостом.
- **Стресс-флаг**: событие помечается stress=true, если сумма gross всех
  SVR-событий в скользящем 24ч-окне вокруг него ≥ **$250k** (крах-день).
- **Разделение подмножеств**: (а) «китовые» = победитель-контракт
  `0xf0570ec48d03171a80ff796dceadf0d385a00004`; (б) «доступные новому входу» =
  все остальные победители. **Вердикт направления — по (б)**; (а) — контекст.
- **Размер**: все события gross ≥ $20k; если их > 50 — топ по gross внутри
  каждого подмножества с целевым сплитом ~15 (а) / ~35 (б). Жёсткий минимум
  для вердикта: n(б) ≥ 20; иначе «выборка недостаточна» → добрать порог вниз
  до $10k gross (только (б)) до достижения n(б) ≥ 20.

## Методология realized proceeds (fork-replay, read-only, $0)

- Состояние — **исторический блок N−1** (до ликвидационной tx: не учитываем
  собственный market-impact победителя и его же свопы в блоке N).
  Агрегаторы историю не котируют → только прямые он-чейн квоты на блоке.
- Квоты: **Uniswap V3 QuoterV2** `quoteExactInput(path, amountIn)` архивным
  eth_call на блоке N−1; пулы НЕ хардкодятся — перебор fee-тиров
  {100, 500, 3000, 10000} через Factory `getPool`; маршруты-кандидаты:
  [coll→debt], [coll→WETH→debt], [coll→WETH→USDC→debt]; берётся максимум.
  Для stETH-семейства дополнительно канонический выход: wstETH→stETH по
  курсу Lido (view на блоке) → **Curve stETH/ETH `get_dy`** на блоке;
  берётся max(UniV3, Curve). Адреса QuoterV2/Factory/Curve — по два источника.
- Proceeds конвертируются в единицы долгового актива (если маршрут кончается
  не в долге — последний лег в долг); realized_net считается в USD по ценам
  AaveOracle на блоке события (та же база, что gross — сопоставимо).
- Мульти-collateral tx: каждый LiquidationCall котируется отдельно, суммируется.
- Событие без рабочего маршрута (нет пула/квота ревертит) — помечается
  `no-route`, исключается из медианы с явным списком в отчёте (не заполняется
  оценкой).
- **Валидация методики до массового прогона**: на 2–3 событиях, где победитель
  свопал атомарно в той же tx, сверить нашу квоту с фактическим token-flow из
  receipt (наблюдаемые фактические proceeds); расхождение объяснить или
  пометить.

## Что фиксируем в отчёте по каждому событию

Дата/блок, победитель (кит/новичок), тип залога, gross(oracle), бид, газ,
swap-proceeds (маршрут), realized net, margin%, stress-флаг, no-route-флаг.
Медиана и p25 отдельно для (а) и (б). Чувствительность: расстояние медианы до
порога; при каком realized-% переворачивается знак EV. Явно незакрытое:
win-rate нового входа НЕ измеряется на форке (только маржа-если-выиграл);
кластеризация 25 «новых входов» по фандингу — если не закрыта, остаётся
residual-риском вердикта.

## Запреты

Read-only, $0: ни транзакций, ни бонда, ни funded-key, ни execution-glue.
Gross не является выводом. Типовые аукционы не мерятся. Base/другие площадки —
вне скоупа этого прогона. Сегодняшняя глубина пулов не подставляется —
только состояние блока N−1.
