# Gate 1 отчёт: SVR-Ethereum realized net — ВЕРДИКТ NO

Дата: 2026-07-06. Pre-registration порога/выборки/методики: `docs/svr_gate1_criteria.md`,
**коммит `1700eb9`** (ДО первой квоты). Скрипт: `analysis/svr_gate1_replay.py`
(+тесты зелёные, stdlib-only). Данные: `data/svr_gate1_sample.json`,
`data/svr_gate1_quotes.json.gz` (+ `_v1_uni_only` и `_patch` — след правки
роутера, см. §4). Всё read-only, $0: ни транзакций, ни бонда — только eth_call-квоты
на исторических блоках N−1.

## Вердикт: **NO** (для нашей модели — read-only атомарный ликвидатор, $0 капитала)

Против pre-registered порога (GO ⇔ медиана realized-net-margin (б) ≥ 40% И p25-net > 0):
- **медиана realized-net-margin (б, исправленный роутер) = 9%** (порог 40%) — dist −31 пп;
- **p25 realized-net (б) = −$45,339** (требуется > 0) — провал второго условия;
- на стресс-хвосте (27 из 34 событий (б)) медиана тоже 9% — не спасает.

Оба условия провалены с запасом. Gate 0 prepare-tooling был на upper-bound (oracle)
марже; realized net её не подтверждает: **оракульный gross реален, но НЕ реализуем
атомарным он-чейн выходом на размерах, где маржа живёт.**

## 1. Выборка (pre-registered, стресс-хвост) и определение маржи

50 событий gross ≥ $20k из кэша Gate 0 (`data/svr_liquidations.json.gz`), 43
стресс-флагнутых (Σgross в 24ч-окне ≥ $250k). Подмножества: **(а) кит
`0xf057…0004` — 15 событий; (б) доступные новому входу — 35** (из них 34 priced +
**1 no-route** — cbBTC-событие без рабочего DEX-выхода, исключено из медиан).
Вердикт — по (б). Залог в (б): WETH (14), wstETH (8), WBTC (4), USDT→WETH (5),
единично cbBTC/weETH/AAVE. Сверка чисел: **50 = 15 (а) + 35 (б)**; в (б) —
**34 priced + 1 no-route**; «25» из Gate 0 — это квалифицированные новые ВХОДЫ
за 12 мес (M2-метрика Gate 0), НЕ размер этой выборки; здесь они не пересчитываются.

**Определение realized-net-margin (цитата критерия, `svr_gate1_criteria.md` §):**
`realized_net = swap-proceeds − погашенный долг − газ(факт из receipt) − SVR-бид
(факт из рефанда билдера)`; `realized-net-margin% = realized_net / gross_oracle`,
где `gross_oracle` = (collateralUSD − debtUSD) по ценам AaveOracle на блоке события
(та же база, что дала prepare-tooling Gate 0). Слиппедж сидит в swap-proceeds.
`ref` (near-spot) — то же, но proceeds = квота клипа 1/20 ×20 (почти-спот-цена).

## 2. Ключевой замер: near-spot vs full-clip realized

| подмножество (б), n=34 priced | p25 | медиана | p75 |
|---|---|---|---|
| **ref near-spot margin** (клип 1/20 ×20 — почти-спот) | 34% | **50%** | 59% |
| **realized full-clip margin** (весь залог одним блоком, лучший он-чейн-путь) | −18% | **9%** | 18% |

- **На near-spot размере залог действительно стоит оракульный gross** (медиана 50%,
  у 22/34 ≥ 40%). Оракульная маржа Gate 0 НЕ фантом — при малом клипе она есть.
- **На полном клипе одним блоком через DEX она испаряется** до 9% медианы. Причина —
  реальный market-impact крупных клипов (см. §3), не бид/газ.

## 3. Где именно рвётся: сегмент по размеру (это и есть суть Gate 1)

| сегмент (б) | n | full-clip margin медиана | realized net медиана | price-impact медиана |
|---|---|---|---|---|
| gross < $150k | 14 | 13% | +$15,890 | 2% |
| gross ≥ $150k | 20 | **−5%** | **−$14,660** | 2% (но хвост до 55%) |

- **Маржа Gate 0 живёт в крупных клипах — ровно там атомарный он-чейн выход рушится.**
  Пример (реальный, верифицирован): 106.5 WBTC ($11.75M залога) → лучший он-чейн
  путь WBTC→USDC→USDT даёт лишь **$8.55M** против $11.24M долга → realized net
  **−$2.7M** (haircut 27% на он-чейн-глубине WBTC). $506k oracle-маржи → −$2.7M
  realized. WETH $10–20M-клипы — тот же механизм (импакт 5–15% даже в глубочайшем пуле).
- **atomic-viable (margin ≥ 40%): всего 3 из 34**, все мелкие ($101–153k gross),
  realized net $47–65k каждое. За 12-мес окно в стресс-хвосте — это не поток бизнеса.
- Суммарный realized net (б, атомарный выход): **−$20.4M**; сумма только положительных
  событий: +$622k. Медиана положительна лишь у мелкого хвоста.

## 4. Честность методики: правка роутера (иначе был бы ложный NO по неверной причине)

Первый прогон (только Uniswap V3, единственный путь) дал медиану full-clip −14% —
но частично это была **неполнота роутера**, не реальный слиппедж. Три дыры найдены
и закрыты (по обязательствам pre-registration), 12 «подозрительных» (margin < −50%)
событий (б) перекотированы:
- **wstETH**: добавлен канонический путь через Curve stETH/ETH (глубокий пул) —
  −1086% → +8%: −1058%/−1293% были ЧИСТО артефактом (тонкие Uni wstETH-пулы).
- **WBTC/BTC**: добавлен маршрут `coll→USDC→debt` (прямой WBTC/USDC 0.3% глубже,
  чем через WETH) — но на реальных $11.75M-клипах даже он даёт 27% импакт = НАСТОЯЩАЯ
  неликвидность, не артефакт.
- fee-тиры: перебор комбинаций по лучшему выходу (не первый существующий пул).

После правки: wstETH-катастрофы исчезли (артефакт), WBTC/cbBTC остались убыточны
(реальный он-чейн-импакт). Медиана (б) сдвинулась −14% → +9% — но порога 40% всё равно
не достигает. **Правка сделала NO честным: он теперь по реальному слиппеджу крупных
клипов, а не по дыре в коде.** Оговорка: full-clip — единственный лучший ПУТЬ без
сплита по венью (нижняя граница); продакшн-агрегатор (1inch/Odos) со сплитом дал бы
лучше, НО $10–20M одним блоком он-чейн фундаментально не выходят в оракул-паритет —
вердикт устойчив к этому ограничению для крупных клипов.

## 5. Почему это NO именно для НАС, и что это значит (факт vs интерпретация)

**Факт (измерен):** realized net на ПУБЛИЧНОМ он-чейн выходе (Uni V3 + Curve, лучший
путь) на размере ликвидации не достигает 40% oracle-gross для (б); маржа
концентрирована в крупных клипах, которые публичным DEX атомарно не выходят.

**Как на самом деле выходит инкумбент (прочитаны tx кита — правит мою прежнюю
гипотезу «варехаусинг»):** кит `0xf057…0004` НЕ держит залог и НЕ несёт направленный
риск. Он атомарно в той же tx отдаёт весь залог одному контрагенту
**`0x51c72848…` (контракт 20,805 байт — приватный MM/RFQ)** и получает стейбл по
**near-spot** — верифицировано: 263.17 WBTC → 25.587M USDT = **$97,225/WBTC**
(≈ оракул), долг гасится флэшлоаном транзитом. Тот же контрагент во 2-м событии
(15,131 wstETH → 0x51c7). Т.е. кит реализует near-spot-маржу (нашу ref, ~50%)
атомарно — но через **приватную MM/RFQ-ликвидность, невидимую публичной DEX-квоте**.

**Почему это NO для нас:** доступ к near-spot-филлу $25M-клипа у `0x51c7`-типа
контрагента = быть известной стороной маркет-мейкеру (RFQ-отношения) ИЛИ иметь
собственный MM-инвентарь = **капитал + private-orderflow/MM-отношения**, что
параметрами программы ИСКЛЮЧЕНО ({read-only, $0, без private orderflow}). Read-only
$0-вход имеет только публичный DEX → измеренные 9% медианы → NO. Наша full-clip
DEX-квота — нижняя граница, НЕ видящая приватную ликвидность; но это ровно та же
граница, что и у $0-игрока, поэтому она — ПРАВИЛЬНАЯ мера для нашей модели, а не
для кита. Барьер, следовательно, — не «неизбежный market-impact» (инкумбент его
обходит приватным филлом), а **доступ к exit-ликвидности крупного залога** =
капитал + MM-отношения (дверь Направления 5 + private-orderflow-параметр).

**Микротест контестируемости (единственный жизнеспособный срез):** 3 atomic-viable
события (margin ≥ 40%, все мелкие $100–153k) выиграны — 0xc7f5… (0.5% доли),
**0x9532… (топ-5 инкумбент, 3.0% $-доли, $1.01M)**, 0x3633… (0.8%). Даже мелкий
рабочий срез инкумбенты контестят (1 из 3 забрал топ-5) → «маржа-если-выиграл»
вдвойне верхняя граница: реальный захват новичком ещё ниже.

## 6. Чувствительность (obligatory)

- Знак: медиана realized 9% vs порог 40% — НЕ пограничное (dist −31 пп), расширение
  выборки вердикт не перевернёт. Порог переворота (медиана→40%) требует, чтобы
  реализация подскочила с 9% до 40% — недостижимо атомарным выходом при измеренном
  импакте крупных клипов.
- Мелкий хвост (gross < $150k) атомарно рабочий (медиана 13%, net+), но абсолютный
  поток ничтожен (3 события ≥40% за год в стресс-выборке) — не бизнес.
- cbBTC (n=1, no-route) — новый актив без глубокого DEX-выхода; помечен, не оценён.

## 7. Явно НЕ измерено / остаточные риски вердикта

- **win-rate НЕ измерялся** (на форке считается только «маржа-если-выиграл»). Даже
  если бы realized проходил порог, доля побед нового входа — отдельный незакрытый
  вопрос. Здесь он вторичен: маржа-если-выиграл уже < порога.
- **Кластеризация 25 «новых входов» по фандинг-графу НЕ проведена** (адрес ≠ сущность).
  Остаётся остаточным риском для тезиса контестируемости Gate 0 — но на вердикт Gate 1
  (realized net) не влияет.
- full-clip = single-path lower bound (см. §4).

## 8. Итог для программы

Gate 0 (upper-bound) → prepare-tooling; **Gate 1 (realized) → NO для {read-only,
$0, публичный DEX-выход}.** SVR-Ethereum остаётся **monitoring**: механизм честный
(аукцион не съедает маржу — подтверждено), маржа реальна и реализуема инкумбентом
(через приватный MM/RFQ near-spot, §5), но её захват требует капитала + private-
orderflow/MM-отношений — вне наших параметров.

**Правка решётки (условная развилка по размеру клипа — цитироваться будет короткая
версия, поэтому формулируем точно):** «аукцион (режим 1) не killer» из Направления 7
Gate 0 верно ТОЛЬКО на крупных клипах, где связывающим становится не аукцион, а
**exit-ликвидность крупного залога** (killer Monad; на публичном DEX — как
market-impact, у инкумбента обойдён приватным филлом = капитал+relationship-барьер).
На МЕЛКИХ клипах публичный выход жизнеспособен (медиана 13%, net+) — и там
связывающим снова становится АУКЦИОН (bid-away) + контестация инкумбентами
(микротест §5: топ-5 инкумбент забрал 1 из 3 жизнеспособных). Т.е. режим 1 не
опровергнут — он развилочный: крупный клип → связывает exit-ликвидность; мелкий
клип → связывает аукцион. Ни один размер не даёт GO для {read-only, $0}.

**Для живого Base-трека (перефрейм Base-прогона):** на Ethereum связывал exit
(клипы $10–20M); Base-тейл — МЕЛКИЕ клипы, где по этому же замеру публичный выход
жизнеспособен → при приходе OEV на Base связывающим станет именно АУКЦИОН (bid-away),
который на Ethereum killer'ом не был. Это меняет Base-вопрос с «есть ли новый edge»
на **defensive economics живого трека**: сколько тейл-маржи под угрозой bid-away.
Передать в monad-liquidator: (i) инвентаризация фидов изолированных рынков
(Chainlink-push = SVR-wrappable vs экзотика = вне угрозы); (ii) bid-away-модель
(при каком биде тейл-маржа → ~0); (iii) увязка с oracle-watcher.
