## Задача A. Направление 8: challenger/fisherman-bounties

### Дискриминатор класса (применяется ДО решётки)

Выигрыш определяется **объективным фактом** (валидный proof побеждает механически)
или **чьим-то голосом/стейком** (DVM-типа)? Если голосом — это P1 (режим 6),
вердикт NO без расхода бюджета. Проверка — по коду challenge-функции, не по докам.

### Обязательные поля карты (Gate 0, по каждому инстансу)

Детерминизм; permissionless (роль/whitelist/KYC); форма конкуренции
(first-valid-proof-WTA / shared-pro-rata / случайная) — ПО КОДУ; награда (доля
slashed-stake / фикс; номинал; ликвидность); bond challenger'а ($-оценка);
exit награды. δ (роллап fault-proofs OP/BOLD): картировать кратко; если bond
> $10k подтверждён — НЕ углубляться (форма-подтверждение, не наш размер).

### Gate 1 (порядок измерений — по решающести)

1. **Частота** (главное): он-чейн скан slashing/challenge-контрактов живых
   инстансов с даты запуска — число РЕАЛЬНЫХ фолтов с выплаченной challenger-
   наградой. Абсолют, по-AVS, помесячно. Окно: вся история (slashing живёт
   с 04.2025, жёсткий минимум для вывода — 6 мес наблюдаемой истории).
2. Bond: ≤$10k (наш unlock) или >$10k (режим капитала).
3. Форма конкуренции — по коду challenge-функции.
4. EV = частота × награда − стоимость тулинга. Санити: EV ≤ суммарного
   slashed-stake за период.

### Классификация пустоты (обязательна, факт отдельно от причины)

Если событий ≈ 0, причина выбирается ДАННЫМИ, не мнением:
- **(а) ранняя стадия / внимание не пришло** — свидетельства: большинство AVS
  держат slashing paused/не включён; есть зафиксированные фолты БЕЗ поданного
  challenge (упущенные баунти = дыра тулинга); TVS/операторы растут.
- **(б) пусто по дизайну (детеррент)** — свидетельства: slashing включён и
  давно, фолты технически детектируемы, но нарушений нет (CoC≫PfC аргумент
  с числами stake vs выгода нарушения); зрелые аналоги (годы) с нулём событий.
- Если свидетельств недостаточно для выбора — писать «не классифицировано»,
  вердикт не выше monitoring.

### Пороги вердикта A

Издержки (база): инфра watcher'а $100–300/мес + интеграция 1–4 недели
(амортизация 6 мес); стоимость капитала бонда 10%/год.
- **NO**: ни один инстанс не проходит конъюнкцию {детерминизм ∧ permissionless
  ∧ bond ≤ $10k ∧ (не-WTA-гонка ИЛИ WTA с ≤3 наблюдаемыми watcher'ами)} — ИЛИ
  частота×награда < издержек при пустоте, классифицированной как (б) дизайн.
- **monitoring**: конъюнкция структурно проходит, но событий ≈ 0 и пустота
  классифицирована как (а) ранняя стадия ИЛИ не классифицирована. Обязателен
  датчик (какое событие переоткрывает) и каденс.
- **prepare-tooling**: ≥1 инстанс с конъюнкцией ∧ ≥1 РЕАЛЬНАЯ выплаченная
  challenger-награда за последние 6 мес ∧ консервативный EV > $500/мес.
- **pilot-ready**: prepare-tooling ∧ ≥3 выплаченных события за 6 мес ∧
  консервативный EV ≥ $2k/мес ∧ exit награды ликвиден.

## Задача B. Morpho V2 (расширение живого трека, не новое направление)

### Gate 0 — механика ТОЛЬКО по коду/спеке (контракты, верифицированный исходник)

Обязательные ответы: тип события (health-based / maturity-default / оба);
механизм закрытия (fixed-spread instant / аукцион / Dutch / иное) — по коду;
параметр бонуса ликвидатора — число из контракта; оракулы живых V2-рынков
(OEV-совместимость: SVR/Atom = маркер изъятия); exit залога; статус деплоя
(какие контракты на каких чейнах, есть ли реальный borrow-поток).
Если исходник не верифицирован/не найден — фиксировать как ограничение.

### Пороги вердикта B (слепок Base-тейла, пороги — как в HyperEVM-замере)

- **NO**: механизм закрытия = аукцион (режим 1); ИЛИ maturity-события
  календарно-публичны И на живых рынках уже >3 уникальных ликвидаторов на
  реальном потоке (профи пришли тем же календарём); ИЛИ exit залога нерабочий.
- **monitoring**: механика не-аукционная, но V2-рынки не живы / borrow-поток
  тонкий (суммарно < $10M активных займов V2 на всех чейнах) / событий
  ликвидации ещё не было. Датчик: рост потока + первые события.
- **prepare-tooling**: не-аукционная механика ∧ живой поток (≥1 рынок с
  ликвидациями ≥ $5k/90д) ∧ ≤3 уникальных ликвидатора на таком рынке ∧
  рабочий exit. (Как Base-тейл: транзиторное окно первого тулинга.)
- **pilot-ready**: НЕ присваивается из этой разведки (это расширение трека
  monad-liquidator — решение там, после своего Gate 1 с fork-replay).

### Gate 1 B (если Gate 0 не убил)

Он-чейн замер по образцу hyperlend_liq.py: topic0 события ликвидации/дефолта
V2 offline-keccak, скан деплойнутых контрактов, кэш data/, тесты. Метрики:
n событий, $-поток/период, уникальные ликвидаторы/рынок, реализованный бонус,
доля maturity-vs-health.

### Передача в monad-liquidator (независимо от вердикта)

Maturity/rollover-мониторинг как расширение бота; смена оракула на V2-рынках
= ранний сигнал OEV-изъятия.
