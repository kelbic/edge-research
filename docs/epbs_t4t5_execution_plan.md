(pre-registration `docs/epbs_gate0_preregistration.md` @ `e2a5d38…`; бэклог

тестнета (триггер S2) осмысленны только **T4 (офлайн-пересчёт, ~1 день)** и **T5 (датчики,

не расширение скоупа.

## 1. T4 — пересчёт EV/газ-моделей под H5 (~1 день, офлайн, stdlib-only)

Цель по бэклогу: чистые функции + юнит-тесты; «реплей-верификация на тестнете = pre-registered
тест H5» (подтверждено, если измеренная дельта ≥10% хотя бы на одном паттерне).

**Вход — библиотека tx-паттернов** (те же, что в pre-registered тесте):
- ликвидация ~300k exec + calldata ~800B (типовой прямой путь);
- своп-пути calldata 800B / 5KiB / 20KiB (компактный роутинг vs агрегаторные KyberSwap/1inch);
- кейс «эксекьютор впервые получает токен X» (новый storage-слот);
- (ветвь CFI) те же паттерны при 2780 (интринсик ↓) и 8038 (state-access ↑).

**Функции** (параметры EIP — конфиг, не константы: числа могут меняться до заморозки):
- `floor_7976(calldata_bytes)` — floor 64 gas/байт uniform (zero==nonzero); правило headroom
  (лимит tx обязан покрывать floor);
- `surcharge_8037(new_slots, new_accounts)` — слот 20k→97.9k (~4.9×), аккаунт 25k→183.6k;
- `al_rule_7981()` — access-list как оптимизация мертва (+1,848 к холодному); правило «дроп AL»;
- `tx_cost(pattern, params)` — итоговая стоимость до/после; `delta%` по паттерну.

**Выход**:
- таблица дельт по паттернам (ожидание из Gate 0: прямой путь ~0%; 5KiB почти не биндится;
  **20KiB ≈ +150%**; новый слот +77.9k — лечится пре-сидом);
- свод операционных правил для `monad-liquidator` mainnet-USDe трека (пересчитать ДО активации;
  Base не трогать до OP-stack-форка): (i) предпочесть компактный роутинг, толстые
  агрегаторные calldata-пути на L1 исключить из выбора маршрута; (ii) пре-сид балансов/слотов
  эксекьютора по целевым токенам (операционная дисциплина, не капитал); (iii) выкинуть
  access-lists (~+0.7%); (iv) headroom-проверка лимита tx против floor;
- юнит-тесты stdlib; файл `analysis/epbs_gas_model.py` + `analysis/test_epbs_gas_model.py`
  (имена — предложение; финально при написании кода, которое требует OK пользователя по
  правилу «код после ревью Gate 0»).

## 3. Явные не-цели (повторение запретов, чтобы план не расползся)

- T1–T3 не пишутся до S2 + OK пользователя; собственная нода — только с разрешения (инфра-шаг).
- Никакого капитала, live-шагов, day-one-ставок (запрет «GO с капиталом» — pre-registration;
  эмпирика H6: day-one alpha прошлых форков — эксплойтерам или никому).
- Base-стек не трогаем до соответствующего OP-stack-форка.
- Прогнозы дат не делаем: реалистичный коридор из Gate 0 (сен–дек 2026, Q4-риск) не уточняем —
  pre-registered тесты привязаны к тестнет-активации, не к календарю.

**Бюджет: ~1.5 дня суммарно** (T4 ~1 день + T5 ~0.5 дня). EV опциона: защита экономики живого
mainnet-трека (единственный уже-доказанный +EV актив программы) + готовность H2-замера к S2.
