## Рамка и отношение к прежнему вердикту C1

Направление 3 / батч C уже дало **NO** по C1 (OEV-аукционы) на Gate 0 — но по
вторичным пересказам (recapture 80–90%, «permissionless-тейл закрылся»), без
первичных данных. Это направление — переоткрытие C1 с первичными данными.
Фиксируем ДО замера: прежний NO **не является** аргументом ни в какую сторону;
вердикт выносится только по метрикам ниже. Если первичные данные подтвердят
вторичные пересказы — это усилит C1-NO; если опровергнут — C1 пересматривается.

**H0:** на OEV-площадках остаётся захватываемая новым участником маржа, потому что
конкуренция идёт по качеству EV-модели, а не по инфраструктуре (наша сила).
**Анти-H0 (проверить первой):** та же олигополия сёрчеров в новой обёртке; бид
съедает маржу до нуля by design (аукцион спроектирован отдавать value протоколу).

**Разрешённые вердикты по каждой площадке: NO / monitoring / prepare-tooling /
pilot-ready.** «Pilot с капиталом» из этой разведки НЕ назначается — только
рекомендация со ссылкой на выполненные критерии. Любой live-шаг — отдельное
решение пользователя.

## Метрики (считаются по каждой площадке раздельно)

Целевое окно ≥ 6–12 мес; жёсткий минимум 3 мес. Если доступно < 3 мес истории —
метрика «данных недостаточно», площадка не может получить выше **monitoring**.

- **M1 Концентрация побед**: доля топ-1 / топ-3 / топ-10 адресов-победителей
  (по числу аукционов И по $-объёму — расхождение фиксировать), HHI, помесячно.
- **M2 Контестируемость**: «новый вход» = адрес-победитель, чья ПЕРВАЯ победа
  на площадке — в последних 12 мес, и который в каком-либо скользящем 3-месячном
  окне после входа набрал **≥ 5%** побед (по числу ИЛИ по $-объёму).
  Кластеризация адресов (funding/поведение) — если очевидна, объединять и
  помечать; не строить кластеризацию сложнее очевидной.
- **M3 Остаточная маржа победителя**: on-chain реконструкция
  `gross ликвидационный профит − бид − газ` по-аукционно; распределение
  (медиана, p25/p75), помесячная динамика, доля топ-5 аукционов в суммарной марже.
- **M4 Поток**: аукционов/мес × медианный gross-размер; отдельно — поднабор
  «наша поверхность» = лендинг-ликвидации на EVM-чейнах (Morpho/Aave/Compound-класс,
  где наш EV-стек применим с доработкой ≤ ~2–4 недель).
- **M5 Санити**: по выборке аукционов `бид + маржа + газ ≈ gross`; расхождение
  медианно ≤ 20% ИЛИ объяснено; суммы за период сверить с внешним агрегатом
  (инцентивы Aave и т.п.) по порядку величины.

## Правила вывода win-rate (НЕ постулировать)

Интервал win-rate нового участника выводится из M2:
- нижняя граница = медианная доля побед квалифицированных новых входов в их
  первые 6 мес (если входов нет — 0);
- верхняя граница = максимальная устойчивая (3-мес окно) доля, достигнутая
  новым входом за 12 мес;
- если квалифицированных новых входов НЕТ — интервал [0; 1%], и это
  одновременно срабатывание K1.

## Издержки (базовые допущения, уточняются в Шаге 1 фактами)

- Инфра: $100–300/мес (существующий стек: VPS, публичные RPC).
- Интеграция площадки + доработка EV-моделей под чужой протокол: 2–6 недель
  работы; амортизируем на 6 мес.
- Депозиты/бонды/бридж — по фактам Шага 1; стоимость капитала 10%/год.
- EV/мес = (поток$ «нашей поверхности» × win-rate × медианная маржа%) − издержки.
  Консервативный EV = нижний край win-rate × медиана маржи; оптимистичный =
  верхний край × p75.

## Killer-критерии (каждый самодостаточен; замер, не мнение)

- **K1 олигополия**: топ-3 ≥ **85%** $-объёма побед устойчиво (каждый из
  последних 6 мес) И ноль квалифицированных новых входов (M2) за 12 мес.
- **K2 маржа→0 by design**: медианная остаточная маржа M3 ≤ 0, ИЛИ
  оптимистичный EV/мес < **$300**.
- **K3 несовпадение поверхности**: «наша поверхность» (M4) < **20%** $-потока
  площадки И абсолютный поток нашей поверхности < **$50k gross/мес** —
  т.е. даже идеальный захват лендинг-остатка не окупает интеграцию.
- **K4 relay-риск / непрозрачность**: историю аукционов получить нельзя
  (off-chain, не публикуется) → это свойство площадки; вердикт ≤ monitoring,
  НЕ заполнять правдоподобной оценкой.
- **K5 latency-бар**: длительность аукциона (окно на расчёт EV + подачу бида)
  < **500 мс** → fail (наш замеренный цикл реакции 166–700 мс, monad-liquidator
  §17); 0.5–2 с → пограничное (пометить); > 2 с → проходим.

## Пороги вердиктов (по площадке)

- **NO**: подтверждён K1, ИЛИ K2, ИЛИ K3, ИЛИ K5-fail.
- **monitoring**: K4 (данных нет / < 3 мес), ИЛИ EV пограничное — знак
  консервативного EV/мес меняется внутри выведенного интервала win-rate × маржи.
- **prepare-tooling** (все одновременно):
  1. контестируемость: ≥ 1 квалифицированный новый вход (M2) за 12 мес;
  2. медианная остаточная маржа > 0;
  3. консервативный EV/мес ≥ **$500**;
  4. вход фактически permissionless (проверено фактом: регистрация/депозит без
     KYC/партнёрства/вайтлиста);
  5. K5 пройден (≥ 0.5 с, с пометкой если < 2 с).
- **pilot-ready**: всё из prepare-tooling И консервативный EV/мес ≥ **$2,000**
  И поток «нашей поверхности» ≥ **50 аукционов/мес** И маржа не фат-тейл
  (топ-5 аукционов < 50% суммарной маржи за окно) И помесячная маржа не в
  систематическом падении (последние 3 мес не хуже −50% от медианы окна).

## Приоритет источников данных (фиксируется)

1. **API3 OEV Network** — он-чейн zk-rollup: полный replay своими скриптами
   (высшее качество).
2. **SVR** — MEV-Share/Flashbots данные, Dune, отчёт LlamaRisk; он-чейн

Вторичные цифры (в т.ч. «95% аукционов за 1–2 блока») либо сверяются с
первоисточником, либо в отчёте помечены «НЕ СВЕРЕНО».
